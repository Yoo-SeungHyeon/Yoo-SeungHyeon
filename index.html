CASE STUDY 01

PROMTREE

"흩어진 문서에서 지식을 모으다"

2025.10 - 11 · 6인 팀 · 삼성전자 생산기술연구소 연계

GitHub →

01. 문제 정의

수천 개의 PDF, 각기 다른 형식, 그리고 찾을 수 없는 정보

삼성전자 생산기술연구소의 엔지니어들은 물성 예측 모델을 개발하기 위해 대용량의 데이터셋이 필요했습니다. 하지만 원천 자료는 수천 개의 PDF 문서에 흩어져 있었고, 문서마다 형식이 달랐습니다.

테이블 구조도 제각각이고, 같은 정보라도 위치가 다르고, 일부는 이미지로 삽입되어 있었습니다. 수작업으로 데이터를 추출하는 것은 시간과 비용 면에서 현실적이지 않았습니다.

[이미지 영역] 다양한 형식의 PDF 문서 예시
형식이 제각각인 문서들의 스크린샷

02. 고민과 접근

"기존 Parser로는 왜 안 되는 걸까?"

처음에는 기존의 PDF 파싱 라이브러리들을 시도했습니다. PyPDF2, pdfplumber, Tabula... 하지만 모두 한계가 있었습니다.

문제는 PDF 표준(ISO 32000)의 복잡성에 있었습니다. PDF는 "보여주는 것"에 최적화된 포맷이지, "데이터 추출"을 위한 포맷이 아니었습니다.

AI와의 협업

Claude와 함께 PDF 표준 문서를 분석했습니다. "정규표현식으로 테이블을 찾는 건 왜 실패하는가?"라는 질문에서 시작해, 좌표 기반 접근법이라는 새로운 방향을 찾았습니다.

텍스트의 위치(x, y 좌표)를 분석해서 같은 행/열에 있는 요소들을 그룹핑하는 방식. 이 아이디어가 자체 Parser 개발의 출발점이 되었습니다.

[이미지 영역] AI와의 대화 스크린샷

[이미지 영역] 좌표 기반 파싱 개념도

03. 시행착오

95%에서 99%로 가는 길이 가장 멀었다

초기 버전은 약 90%의 정확도를 보였습니다. 하지만 삼성전자의 요구사항은 99% 이상이었습니다. 나머지 10%는 예외 케이스들의 집합이었고, 하나를 해결하면 다른 문제가 생겼습니다.

시도했던 것들

시도 1: 규칙 기반 후처리 → 예외 케이스가 너무 많아 유지보수 불가능

시도 2: Few-shot 프롬프팅 → 일관성 부족

시도 3: 표 이미지 → OCR → 구조화 → 속도 문제

최종: 좌표 분석 + Document Layout Model + 검증 레이어 조합

[이미지 영역] 정확도 개선 과정 그래프

04. 해결

자체 Parser와 RAG 기반 챗봇

최종적으로 구축한 시스템은 세 가지 핵심 컴포넌트로 구성됩니다:

1. 자체 PDF Parser: PDF를 마크다운으로 변환. 좌표 분석과 Document Layout Model을 결합해 테이블과 텍스트를 정확하게 구조화합니다.

2. Extract RAG: 마크다운에서 필요한 물성 정보만 추출. 규격화된 형태로 데이터베이스에 저장합니다.

3. 검색 챗봇: FastAPI + Qdrant 기반. 엔지니어들이 자연어로 물성 정보를 검색할 수 있게 했습니다.

[이미지 영역] 시스템 아키텍처

[이미지 영역] 챗봇 UI

LEARNING

"완벽한 자동화보다, 신뢰할 수 있는 반자동화가 더 가치 있습니다. 99%의 자동화와 1%의 인간 검증이 100%의 자동화보다 현실적이고, 사용자가 신뢰할 수 있는 시스템이 결국 더 오래 쓰입니다."

CASE STUDY 02

DAILYPET

"일상의 데이터에서 건강의 신호를 읽다"

2025.09 · 6인 팀 · SSAFY 특화 프로젝트 우수상 (2등)

GitHub →

01. 문제 정의

반려동물의 건강 이상, 왜 항상 뒤늦게 알게 될까?

반려동물은 말을 하지 못합니다. 아파도 숨기는 경향이 있고, 보호자가 이상을 느꼈을 때는 이미 상태가 악화된 경우가 많습니다.

매일 기록하는 산책 시간, 식사량, 수면 패턴... 이 일상 데이터에서 건강 이상의 조기 신호를 찾을 수 있지 않을까?

[이미지 영역] 문제 상황 일러스트

02. 고민과 접근

"어떤 모델로 '이상'을 정의할 것인가?"

이상치 탐지(Anomaly Detection)를 적용하기로 했지만, 어떤 모델을 선택할지가 문제였습니다. Autoencoder? Isolation Forest? One-Class SVM?

특히 Cold Start 문제가 고민이었습니다. 새로 가입한 사용자는 데이터가 없는데, 어떻게 "평소"를 정의할 수 있을까?

AI와의 협업

여러 이상치 탐지 모델을 비교 분석하면서, 각 모델의 특성과 우리 데이터의 특성을 매칭했습니다.

결론: Isolation Forest + IQR 하이브리드. 적은 데이터로도 동작하고(Cold Start 대응), 추론 속도가 빨라 실시간 알림에 적합했습니다.

[이미지 영역] 모델 비교 분석표

03. 시행착오

False Positive와의 싸움

초기 모델은 너무 민감했습니다. 주말에 산책을 조금 덜 했다고 "이상 감지" 알림이 울렸고, 사용자들은 금방 알림을 꺼버렸습니다.

개선 과정

문제: 단일 지표 이상 → 즉시 알림 → 피로도 증가

개선: 복합 지표 가중치 + 연속성 체크 + 심각도 등급화

04. 해결

일상 기록 → 건강 모니터링 → 맞춤 추천

단순한 이상 탐지를 넘어, 수집된 데이터를 기반으로 펫 금융상품(보험) 추천까지 연결했습니다.

반려동물의 품종, 나이, 활동량, 건강 기록을 분석해 적합한 보험 상품을 RAG 기반으로 추천합니다.

[이미지 영역] 앱 UI

[이미지 영역] 이상 탐지 결과

LEARNING

"좋은 AI 모델보다 좋은 데이터 설계가 먼저입니다. 어떤 데이터를 어떻게 수집할지, 수집 단계에서의 설계가 전체 시스템의 성능 상한선을 결정합니다."

CASE STUDY 03

CDD

"사람이 갈 수 없는 곳의 눈이 되다"

2025.07 - 08 · 6인 팀 · SSAFY 공통 프로젝트 우수상 (3등)

GitHub →

01. 문제 정의

고층 건물의 균열, 어떻게 검사할 것인가?

고층 건물, 교량 하부, 댐 벽면... 대형 구조물의 균열 점검은 위험하고 비용이 많이 듭니다.

드론과 AI를 결합하면, 안전하고 빠르게 정밀 점검을 할 수 있지 않을까?

[이미지 영역] 문제 상황 사진

02. 고민과 접근

"클라우드로 보내면 늦는다"

처음 계획은 단순했습니다. 드론이 촬영 → 서버로 전송 → AI 분석 → 결과 반환. 하지만 실시간 점검이 필요한 현장에서 네트워크 지연은 치명적이었습니다.

결국 온디바이스 AI로 방향을 틀었습니다.

기술적 도전

문제는 영상 전송이었습니다. WebRTC나 RTSP의 H.264 코덱은 하드웨어 인코더가 필요한데, Jetson Orin Nano에서는 지원하지 않았습니다.

해결책: GPU 기반 MJPEG 인코딩 + RTSP 전송. 50ms 지연, 최대 50fps의 실시간 전송을 달성했습니다.

[이미지 영역] 영상 전송 방식 비교

03. 시행착오

LiDAR 연동, 문서가 없다

균열의 깊이를 측정하기 위해 LiDAR(X4-Pro)를 도입했는데, 공식 SDK가 우리 환경을 지원하지 않았습니다.

해결 과정

제조사 GitHub을 샅샅이 뒤졌습니다. ROS2 Driver 저장소에서 필요한 파라미터 값들을 찾아냈고, SDK를 직접 수정해서 연동에 성공했습니다.

공식 문서가 없으면, 코드가 문서다.

04. 해결

드론이 보고, AI가 분석하고, 3D로 기록한다

1. 실시간 탐지: YOLO 기반 온디바이스 AI가 균열을 감지

2. 깊이 측정: LiDAR와 Segmentation 모델로 균열의 3D 정보 추출

3. 기록 및 공유: 녹화 영상을 3D 렌더링하고 웹으로 공유

[이미지 영역] 실시간 탐지 화면

[이미지 영역] 3D 렌더링 결과
